Personal Project · January 2026 – February 2026

Python Banking & ATM System

Full console-based banking system with persistent storage, security checks, and an admin panel.

Built an end-to-end ATM and banking system that lets users open accounts, receive debit cards, deposit, withdraw, check balances, and includes an admin interface for managing and unlocking accounts.

  • Generated unique account IDs and PINs by creating random numbers and checking for duplicates to avoid collisions.
  • Maintained accurate balances, PINs, and lock status by storing all accounts in a list and updating it on every operation.
  • Implemented persistent storage using a save_accounts() function that rewrites the accounts.txt file after each change.
  • Added PIN security with 3 attempts, automatically locking cards and updating the file when limits were exceeded, and resetting attempts on success.
  • Designed an admin panel to view all accounts and unlock cards, plus robust input validation to handle invalid choices and edge cases without crashing.

Personal Project · January 2026 – February 2026

Python Bus Booking System

File-backed bus booking and cancellation system with seat tracking and resilient menus.

Developed a Python system that manages bus bookings with unique booking IDs, accurate seat availability, and persistent storage to simulate how real services track users and trips.

  • Ensured every booking had a unique ID by generating random values and checking against existing IDs.
  • Kept seat availability synced per bus by updating booked seats on every booking and cancellation.
  • Used file storage to persist bookings, rebuilding booked seat counts at startup by parsing existing records.
  • Implemented a continuous menu loop for smooth user experience until explicit exit.
  • Handled invalid input (bus numbers, menu options, booking IDs) gracefully and rebuilt the bookings file after cancellations to keep data clean.
