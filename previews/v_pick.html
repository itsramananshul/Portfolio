Four animation stacks.
Same content. Different bones.

Your live index.html isn't touched. Each variant below is a standalone preview using a different animation library/approach. Open each on your phone, see what you like.

v1
— motion one (vanilla framer-motion)
Restrained, native, fast.
Vanilla JS animations via Motion One — same author as framer-motion, same easing curves, no React needed. Word-by-word stagger reveals on hero + section headings, scroll-triggered fades on stat cards and project rows. Clean dark theme with a cyan accent. Loads in <30kb of JS.
↗ v2
— gsap + scrolltrigger + lenis
Awwwards-mode. Maximum motion.
GSAP timeline-based hero entrance, infinite marquee strip between sections, smooth Lenis scrolling, parallax background numbers, custom red cursor with magnetic hover effect, project rows that expand on hover. Bebas Neue display + Instrument Serif italic. The "I want to win design awards" treatment.
↗ v3
— react + framer-motion (esm, no build)
The actual framer-motion library.
React 18 + framer-motion 11 loaded directly via ESM CDN — runs in the browser without any build step. motion.div primitives with whileHover, whileTap, useInView, variants with stagger. Mesh-gradient backdrop, glass cards, gradient pill nav. The closest thing to your "what if it was a real React app".
↗ v4
— 21st.dev inspired (magic ui · bento · glass)
Premium component-library aesthetic.
The 21st.dev / shadcn / magic-ui visual language: floating pill nav with animated logo mark, shine-effect tag pill, conic-gradient borders that animate on hover, bento grid (4-col + 2-col asymmetric), purple/pink/cyan beam color palette, glass-morphism. Geist font. The closest "designer template" feel without React.
↗

All four are standalone — no install needed, no build step, all run in the browser. Same content (Sophomore + UC, OpenPrem Founding Engineer, ARIA + Auriva + VoxVitals + GradeIntel). Tell me which feels right and I'll commit it as the new index.html.
